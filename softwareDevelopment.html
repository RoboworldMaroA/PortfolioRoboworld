SmartLuggage – The Application for Easy Packing

The goal is to build an application that:
• is easy to use and fast to run
• take advantage of the latest technology available
• machine learning for object recognition will be applied as an innovative part of this project

Why the SmartLuggage app will make a traveller's life easier?
• less stress of forgetting an important document needed during the trip
• making your own packing list means, you always have to rely on your memory, with the SmartLuggage it will be easier because will give you a hint what to pack for your trip
• by choosing a destination and type of travel, the application will show you what to pack to avoid carrying unnecessary items during your trip
• your travel list is always with you on your phone
• the approximate weight of your luggage, given after creating the list of items, will protect you from excess baggage at the airport
• SmartLuggage is designed to make packing easy, fast and pleasant from now on

Conclusion

Built app makes packing easy through an easy-to-use smart item allocation system. It allocates items to the appropriate category based on the user's activity and gender. It calculates the weight and number of items in the packed luggage. It allows you to add your own items and delete unnecessary ones. The colors in the app are well thought out and consistent. Google Map API is used to search destination, it allows to find practically every address in the world that is in Google database. The application is intuitive and uses the latest technology. If in the future it would be possible to implement the vision system with the main application and put it on the server, one would get the impression that it is a professionally built application. However, there are a few elements that have not been implemented to perfection, such as the secure login system, weather and payments.

Future Scope of Project

In the future, the video system should be improved. Functionalities such as weather recognition depending on the season and direction of travel should be added. It should also be possible to save the list and send it to an e-mail address or via Messenger.

Vision System


According to the project requirements as an optional requirement is ability to add user items by using the camera. Unfortunately, it was not possible to integrate the vision system into the application. The prototype application was built on an Ubuntu system and written in Python using Open CV libraries and SIFT functions. At the moment it is a separate application. It can be integrated into the main system in the next stage of the project. Application contains in the memory objects which the system recognises and displays on the screen information what was found. I build training model that allows add an object to the system. System asking the user to take a photo, then the photo is stored in memory, and in the next phase the application is opened and can recognise whether the object is in the database or not.

Setup Environment

I used the instructions posted on pyimagesearch.com to install all dependences.
The environment installed on my system is:
Python 3.8.2
GCC 9.3.0
CV2 3.4.4

To run the application, I created a Bash file to enable the commands stored in the file.

The largest window is our main window. In this window is smaller blue square is displayed. This is where we have to place the object that we want to recognize. The smaller window on the left shows only what we see in this blue window.

Press the space bar to confirm that you want to take the photo. New window will appear with the photo that just have been taken. The system will ask you if you are happy with the result. If so confirm by pressing esc or if not press space again and choose another frame.

After pressing esc, the image will be saved to the hard drive. Another program will be started which allows the recognition of the objects.

The other two objects stored in the database are a G-Shock watch and a pack of Oral-B toothpaste. They are displayed on the screen on the top left corner If you want to test the app, you can open the photo stored in the app and take a picture with your phone, then show the photo instead of the object - in this case the app works much more accurately.
